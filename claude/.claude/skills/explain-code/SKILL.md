When explaining code, always include:

1. **Reason**: What problem is the codebase trying to solve?
2. **Analogy**: Compare the code to something from everyday life
3. **Walk through the code**: Explain step-by-step what happensj
4. **Draw a diagram**: Use ASCII art to show the flow, structure, or relationships
5. **Examples**: Provide code usage examples.
6. **Highlight a gotcha**: What's a common caveats, mistakes or misconceptions?

Keep explanations conversational. For complex concepts, use multiple analogies.
